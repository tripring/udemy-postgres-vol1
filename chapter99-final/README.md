## 環境のクリーンアップ

学習が終わったら、Dockerのリソースを整理しましょう。

### 共通DB環境

```bash
# コンテナを停止（データは残る）
docker compose down

# コンテナを停止してデータ（ボリューム）も削除
docker compose down -v

# コンテナ・データ・イメージをすべて削除
docker compose down -v --rmi all
```

### Dockerの未使用リソースをまとめて削除（強力）

```bash
# 停止中のコンテナ・未使用イメージ・未使用ボリューム・キャッシュを一括削除
# ※ このコース以外のDockerリソースも削除されるので注意
docker system prune -a --volumes
```

> **注意**: `docker system prune -a --volumes` は他のプロジェクトのDockerイメージ・ボリュームも削除します。このコース専用のリソースだけを消したい場合は、上の `docker compose down -v --rmi all` を使ってください。

---

### 各コマンドの意味

| オプション | 意味 |
|---|---|
| `down` | コンテナを停止・削除 |
| `-v` / `--volumes` | データボリュームも削除（DBのデータが消える） |
| `--rmi all` | 使用したDockerイメージも削除（ディスクを大きく解放できる） |
| `docker system prune` | 未使用のリソースをまとめて削除 |

## Vol.2 へのステップ

Vol.1 ではクエリの書き方・チューニング・トランザクション・パーティショニングを扱いました。

**Vol.2（運用・インフラ設計編）** では、以下のテーマを扱います。

| 章 | テーマ | 内容 |
|---|---|---|
| 00 | セットアップ | Vol.2 環境の構築 |
| 01 | トリガー・プロシージャ | DBレイヤーでの自動化とエラーハンドリング |
| 02 | バックアップ | pg_dump/pg_restore・PITR概念 |
| 03 | レプリケーション | Primary/Replica構成と読み取り負荷分散 |
| 04 | VACUUM | テーブル膨張の診断・自動メンテナンス設計・XIDラップアラウンド |
| 05 | 接続プーリング | PgBouncerによる接続数管理と大規模対応 |
| 06 | 監査・セキュリティ | ロール設計・監査ログ・最小権限の原則 |
